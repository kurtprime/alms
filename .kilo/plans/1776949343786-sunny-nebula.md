# Plan: Teacher Quiz Management Dashboard (`/teacher/quizzes`)

## Overview

Create an interactive `/teacher/quizzes` page that serves as a central hub for teachers to view, filter, and manage all their quizzes across all classes. The page navigates to the existing Quiz Builder (`CreateQuiz`) for editing individual quizzes.

## Key Discovery

`adminProcedure` in `src/trpc/init.ts:37-61` already allows **both admin and teacher roles**, so all existing admin quiz procedures (`addQuizQuestion`, `getQuizQuestions`, `deleteQuestion`, etc.) are already accessible to teachers. No new auth layer is needed for quiz editing.

---

## Files to Create/Modify

### 1. NEW: `src/modules/user/server/procedures/quiz.ts` — tRPC procedures

A new procedure file with `protectedProcedure` queries for the teacher quiz dashboard:

**`getTeacherQuizzes`** — Fetches all quizzes for the logged-in teacher:

- Query chain: `classSubjects (teacherId = userId) → lessons → lessonType (type='quiz') → quiz`
- For each quiz, join: `quizQuestion` count, `quizAttempt` stats (total submissions, pending grading count), `subjectName` for display
- Returns flat array of quiz objects:
  ```ts
  {
    quizId: number;
    quizName: string | null;
    quizDescription: string | null;
    quizStatus: string; // from quiz.status
    lessonTypeStatus: string; // from lessonType.status (draft/published)
    classSubjectId: string; // for navigation
    className: string; // organization.name
    subjectName: string; // subjectName.name
    questionCount: number;
    totalSubmissions: number; // attempts with status in [submitted, graded, expired]
    pendingGrading: number; // attempts with status in [submitted, expired]
    timeLimit: number | null;
    maxAttempts: number | null;
    startDate: Date | null;
    endDate: Date | null;
    lessonTypeId: number; // for edit navigation
    createdAt: Date;
    updatedAt: Date;
  }
  ```

**`getTeacherQuizStats`** — Aggregate dashboard stats:

- Total quizzes, total questions, total submissions, pending grading count
- Used for summary cards at top of page

- Title: "Quiz Management"
- Subtitle with teacher name

- **Class filter**: Dropdown populated from quiz data's unique `classSubjectId`/`className` values
- **Status filter**: All / Draft / Published tabs or buttons
- Layout: horizontal bar below stats

#### Quiz List (Card Grid — confirmed)

Each quiz card displays:

- Quiz name (fallback "Untitled Quiz")
- Class/Subject badge
- Status badge (Draft = gray, Published = green)
- Question count
- Submissions / Pending grading counts
- Time limit, availability window
- Last updated date
- **Actions**:
  - "Edit Quiz" → navigates to `/class/{classSubjectId}/quiz/e/{quizId}` (existing `CreateQuiz`)

#### Empty State

- When no quizzes exist: illustration + guidance to create one from a class page

#### Loading State

- Skeleton cards matching the grid layout

## Tech Stack & Patterns

- **tRPC**: `protectedProcedure` (not admin — since this is teacher-specific)
- **React Query**: `useSuspenseQuery` for data, `useMutation` for actions
- **UI Components**: shadcn `Card`, `Badge`, `Button`, `Input`, `DropdownMenu`, `Tabs`, `Dialog`
- **Styling**: Tailwind CSS, matching existing dark mode patterns
- **Icons**: lucide-react (already used throughout)
- **State**: Client-side filtering/sorting with `useState`, no URL params needed
- **Auto-save**: Not needed here — quiz editing uses existing `CreateQuiz` + auto-save hooks

---

## What We're NOT Changing

- The existing `CreateQuiz` component and its question type editors
- The existing `QuizSettingsDialog` (we'll reuse it as-is via import)
- The admin procedures (already accessible to teachers via `adminProcedure`)
- The sidebar link (already points to `/teacher/quizzes`)
- The database schema (no migrations needed)
