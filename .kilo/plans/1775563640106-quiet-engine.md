# Horizontal Tabs on Mobile with Vertical Sidebar on Desktop

## Problem

The `orientation="vertical"` prop on the Tabs component applies globally, preventing horizontal tab layout on mobile. The Tabs component doesn't support responsive orientation out of the box.

## Solution

Use two separate Tabs components - one for mobile (horizontal) and one for desktop (vertical), conditionally rendered via Tailwind breakpoints.

## Implementation

### Changes to `src/modules/user/ui/Views/ClassIdView.tsx`

1. **Create two independent Tabs components:**

```tsx
{
  /* Mobile: Horizontal Tabs */
}
<div className="flex md:hidden w-full h-full bg-sidebar/92">
  <Tabs defaultValue="overview" orientation="horizontal" className="flex flex-col h-full w-full">
    <TabsList className="flex flex-row h-auto w-full items-center justify-start rounded-none bg-transparent p-2 gap-1 shrink-0 overflow-x-auto border-b border-white/10">
      {visibleTabs.map(({ value, label, icon: Icon }) => (
        <TabsTrigger
          key={value}
          value={value}
          className={`${baseTriggerClasses} shrink-0 px-3 py-2 text-xs justify-center`}
        >
          <Icon className="h-4 w-4 shrink-0" />
          <span className="truncate">{label}</span>
        </TabsTrigger>
      ))}
    </TabsList>
    {/* Mobile Content - needs separate TabsContent for mobile */}
    <div className="flex-1 min-h-0 overflow-y-auto bg-background">
      <TabsContent value="announcement" className="m-0 h-full focus-visible:outline-none">
        <AnnouncementView classId={classId} session={session} />
      </TabsContent>
      {/* ... other TabsContent */}
    </div>
  </Tabs>
</div>;

{
  /* Desktop: Vertical Tabs */
}
<div className="hidden md:flex md:flex-col md:w-56 md:h-full bg-sidebar/92">
  <TabsList className="flex flex-col h-full w-full items-stretch justify-start rounded-none bg-transparent p-2 gap-1">
    {visibleTabs.map(({ value, label, icon: Icon }) => (
      <TabsTrigger
        key={value}
        value={value}
        className={`${baseTriggerClasses} px-4 py-3 text-sm justify-start`}
      >
        <span className="absolute left-0 top-1/2 -translate-y-1/2 h-6 w-[3px] rounded-r-full bg-transparent transition-colors group-data-[state=active]:bg-red-500" />
        <Icon className="h-5 w-5 shrink-0" />
        {label}
      </TabsTrigger>
    ))}
  </TabsList>
</div>;
```

### Key Considerations

1. **State Synchronization**: The two Tabs components will have independent state. When switching between mobile and desktop, the active tab may reset to default. This may be acceptable or can be addressed later with URL-based state.

2. **Shared Content Issue**: Each Tabs component needs its own TabsContent elements since they can't be shared between two different Tabs roots. This means content is rendered twice in the DOM (though only one is visible at a time).

3. **Alternative - Single Tabs with CSS Override**: Could try to override the CSS on mobile to make TabsList horizontal despite vertical orientation, but this is hacky and may break accessibility/animations.

## Recommended Approach

Use two separate Tabs components as shown above. Accept the state reset when switching breakpoints as a trade-off for cleaner, more maintainable code.
