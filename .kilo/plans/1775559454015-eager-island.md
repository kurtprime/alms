# UX Improvement Plan: ClassIdView Tabs

## Current Issues

1. **Wrapper div is unnecessary** - TabsList should carry all styling directly
2. **Tabs variant not utilized** - shadcn's TabsTrigger has built-in `default` and `line` variants with active indicators
3. **Mobile layout awkward** - Horizontal scrolling tabs on mobile with cramped layout
4. **No gap between sidebar tabs** - Tabs stack directly without spacing
5. **Active tab indicator hard to see** - The `after:` pseudo-element indicator is subtle on dark sidebar
6. **Inconsistent opacity** - Mobile uses `/60` but desktop uses `/90`

## Proposed Changes

### 1. Consolidate TabsList Styling

Remove the wrapper `<div>` and apply all classes directly to `<TabsList>`:

- Keep `md:flex-col md:h-full md:w-56`
- Keep `md:rounded-r-lg`
- Add `md:gap-1` for spacing between vertical tabs
- Use `bg-sidebar/60` consistently (remove `/90`)

### 2. Use the `line` Variant for Active Indicators

Pass `variant="line"` to TabsList. This:

- Removes default `bg-muted` from TabsList
- Shows a vertical indicator bar on the right side of active tabs (built into shadcn via `after:bg-foreground`)
- Better suited for sidebar navigation pattern

### 3. Improve Mobile Experience

On mobile, keep horizontal tabs but:

- Add `overflow-x-auto` for smooth scrolling
- Ensure tabs are evenly spaced with `justify-between` or `gap`
- Add horizontal padding for breathing room

### 4. Add Hover Feedback

Extend hover states via custom classes:

- Inactive tabs: lighten background slightly on hover
- Ensure cursor pointer on clickable tabs

### 5. Clean Up Active State Styling

Override TabsTrigger active state to:

- Use `text-sidebar-foreground` (full contrast) for active tab
- Ensure the indicator bar is visible with proper contrast

## Implementation

```tsx
<TabsList
  variant="line"
  className="h-12 w-full justify-between gap-2 overflow-x-auto bg-sidebar/60 p-4 md:flex-col md:h-full md:w-56 md:justify-start md:gap-1 md:rounded-r-lg md:bg-sidebar/60 md:p-2"
>
  <TabsTrigger
    value="announcement"
    className="data-[active]:text-sidebar-foreground-foreground md:text-base"
  >
    ...
  </TabsTrigger>
  ...
</TabsList>
```
