# Plan: Class Header Image & Edit Feature

## Context

User wants to add to the AnnouncementOverview section:

1. Display a class header image (full-width background)
2. Teacher option to edit/change the header image

**Answers provided:**

- Image scope: Per class subject (classSubjects.id)
- Edit fields: Header image only
- Image source: Upload from device

## Implementation Steps

### Step 1: Add headerImage field to classSubjects table

**File:** `src/db/schemas/subject-schema.ts`

- Add `headerImage` (text, nullable) field to `classSubjects` table

**File:** Database migration (create new migration file)

- Add `header_image` column to `class_subject` table

### Step 2: Add UploadThing endpoint for class header

**File:** `src/services/uploadthing/router.ts`

- Add `classHeaderUploader` endpoint (maxFileSize: '8MB', accepts images only)
- Stores uploaded URL, returns it via onUploadComplete callback
- No DB insert needed (we'll update classSubjects directly via tRPC mutation)

### Step 3: Add tRPC procedures

**File:** `src/modules/user/server/procedures/class.ts`

- Add `getClassSubjectDetails` query - returns classSubjects row including headerImage
- Add `updateClassSubjectHeaderImage` mutation - updates headerImage field

### Step 4: Create ClassHeaderImage component

**File:** `src/modules/user/ui/components/ClassHeaderImage/index.tsx`

```tsx
// Component structure:
// - Full-width div with background image (h-32 md:h-44)
// - Default gradient if no image
// - Teacher sees camera icon overlay on hover
// - Opens dialog to upload/change image
// - Uses useUploadThing("classHeaderUploader")
```

### Step 5: Update AnnouncementOverview

**File:** `src/modules/user/ui/components/AnnouncementOverview/index.tsx`

- Import and render `ClassHeaderImage` at top
- Add edit button (only for teachers) in the header area
- Fetch headerImage via `getClassSubjectDetails` query
- Pass headerImage and onUpdate callback to ClassHeaderImage

### Step 6: Add to ClassIdView (if needed)

**File:** `src/modules/user/ui/Views/ClassIdView.tsx`

- May need to pass classId to AnnouncementOverview for the new query
