# Plan: Vertical Tabs & Collapsed Sidebar for ClassIdView

## Goal

Transform the ClassIdView component to have:

1. **Vertical tabs** instead of horizontal tabs
2. **Sidebar collapsed by default** when viewing `/class/[classId]` routes

---

## Current State Analysis

### Sidebar State Management

- `SidebarProvider` in `src/components/ui/sidebar.tsx` accepts `defaultOpen` prop (line 52)
- Currently set to `true` (expanded by default) in `src/app/(user)/layout.tsx`
- Sidebar state controlled via `useSidebar()` hook's `setOpen` function

### ClassIdView Structure

- Located at `src/modules/user/ui/Views/ClassIdView.tsx`
- Uses Radix UI `Tabs`, `TabsList`, `TabsTrigger`, `TabsContent`
- Currently has **horizontal tabs** with sticky header at top
- 4 tabs: Announcements, Overview, Students, Gradebook (teacher-only)

---

## Implementation Plan

### Step 1: Create a Layout for Class Routes with Collapsed Sidebar

**Problem**: The `(user)/layout.tsx` sets `defaultOpen={true}` for all user routes.

**Solution**: Create a route group layout specifically for class routes that sets `defaultOpen={false}`.

**Files to create/modify**:

- Create `src/app/(user)/class/(classLayout)/layout.tsx` (or modify existing class route structure)

**Note**: We need to restructure the class routes:

- Current: `src/app/(user)/class/[classId]/page.tsx`
- New structure: `src/app/(user)/class/(with-sidebar-control)/[classId]/page.tsx`

Or simpler approach: Use a client-side effect in ClassIdView to collapse the sidebar on mount.

### Step 2: Client-Side Sidebar Control (Recommended - Simpler)

Modify `ClassIdView.tsx` to use a client effect that collapses the sidebar on mount.

**Implementation**:

```tsx
'use client';
// Add at the top of ClassIdView component
useEffect(() => {
  const { setOpen } = useSidebar();
  setOpen(false);
  return () => setOpen(true); // Restore on unmount if navigating away
}, []);
```

**Trade-off**: This requires making ClassIdView a client component or creating a wrapper client component.

### Step 3: Convert Horizontal Tabs to Vertical Tabs

Radix UI `Tabs` component supports `orientation="vertical"` prop.

**Current horizontal structure**:

```tsx
<Tabs defaultValue="overview" className="flex flex-col flex-1 overflow-hidden">
  <div className="sticky top-0 z-20 bg-background border-b">
    <TabsList className="h-12 w-full justify-start rounded-none bg-transparent p-0 px-4">
      {/* Horizontal TabsTrigger items */}
    </TabsList>
  </div>
  <div className="flex-1 overflow-y-auto">
    <TabsContent value="..." />
  </div>
</Tabs>
```

**New vertical structure**:

```tsx
<Tabs defaultValue="overview" orientation="vertical" className="flex flex-1 overflow-hidden">
  <TabsList className="flex-col h-auto w-full justify-start rounded-none bg-transparent border-r">
    {/* Vertical TabsTrigger items */}
  </TabsList>
  <div className="flex-1 overflow-y-auto">
    <TabsContent value="..." />
  </div>
</Tabs>
```

**Styling changes for vertical tabs**:

- Remove `sticky top-0 z-20 bg-background border-b` wrapper
- Change `TabsList` from `h-12 w-full justify-start rounded-none bg-transparent p-0 px-4` to `flex-col h-auto w-[200px] justify-start rounded-none bg-transparent border-r`
- Change each `TabsTrigger` from horizontal border-bottom style to vertical border-right style:
  - Replace `border-b-2 border-transparent` with `border-r-2 border-transparent`
  - Replace `data-[state=active]:border-primary` with `data-[state=active]:border-r-primary`
  - Add `border-l-2 border-transparent` for consistent vertical styling

## Recommended Approach

### For Sidebar: Use Client-Side Wrapper

Create `ClassIdViewWrapper.tsx`:

```tsx
'use client';

import { useSidebar } from '@/components/ui/sidebar';
import { useEffect } from 'react';
import ClassIdView from './ClassIdView';

export default function ClassIdViewWrapper({ classId }: { classId: string }) {
  const { setOpen } = useSidebar();

  useEffect(() => {
    setOpen(false);
    return () => setOpen(true);
  }, [setOpen]);

  return <ClassIdView classId={classId} />;
}
```

Then update `src/app/(user)/class/[classId]/page.tsx` to use the wrapper.

### For Vertical Tabs: Direct Styling Changes

Modify the `TabsList` and `TabsTrigger` components in `ClassIdView.tsx` to use vertical orientation and styling.

## Final Implementation Details

### ClassIdViewWrapper.tsx

```tsx
'use client';

import { useSidebar } from '@/components/ui/sidebar';
import { useEffect } from 'react';
import ClassIdView from './ClassIdView';

export default function ClassIdViewWrapper({ classId }: { classId: string }) {
  const { setOpen } = useSidebar();

  useEffect(() => {
    setOpen(false); // Collapse sidebar on mount
    return () => setOpen(true); // Restore on unmount
  }, [setOpen]);

  return <ClassIdView classId={classId} />;
}
```

### Vertical Tabs Styling

**Desktop (md and above)**:

- `TabsList`: `flex-col h-auto w-[180px] justify-start rounded-none bg-transparent border-r`
- `TabsTrigger`:
  - Default: `border-r-2 border-transparent`
  - Hover: `hover:bg-muted/50 hover:border-r-muted-foreground/50`
  - Active: `data-[state=active]:border-r-primary`

**Mobile (below md)**:

- `TabsList`: `flex-row w-full justify-start rounded-none bg-transparent border-b`
- `TabsTrigger`: `border-b-2 border-transparent` / `data-[state=active]:border-b-primary`

This is achieved using Tailwind's responsive prefixes: `md:flex-col md:w-[180px] md:border-r-2` for the list and `md:border-r-2` / `md:data-[state=active]:border-r-primary` for triggers.
