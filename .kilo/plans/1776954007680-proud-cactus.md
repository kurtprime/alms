# Plan: Replace Inline Document Viewer with Route Navigation

## Goal

When a user clicks on a lesson item (handout/quiz/assignment) in the class Overview tab, instead of opening an inline document viewer panel, redirect them to the dedicated route page (e.g., `/class/[classId]/handout/[lessonTypeId]`).

## Current Behavior

- Clicking a lesson item in `LessonTypeRow` calls `handleClick()` which sets `activeItem` in the `DocumentViewerContext`, opening a resizable side-panel document viewer.
- The layout uses `ResizablePanelGroup` to split the screen between the lesson list and the viewer.
- A mobile document sheet overlays the screen on small viewports.
- The `ViewerToggle` component is already disabled (returns `null`).

## Target Behavior

- Clicking a lesson item navigates to its dedicated route page via `router.push()`.
- The inline document viewer, resizable panels, mobile sheet, and viewer context are removed.
- The Overview tab becomes a simple single-column lesson list.

---

## Changes

### 1. `lesson-type-row.tsx` — Replace viewer logic with navigation

- Remove imports: `useDocumentViewer` from `./context`
- Remove destructured values: `activeItem`, `setActiveItem`, `toggleViewer`, `isViewerOpen`
- Remove `isActive` computed value (used for the viewer highlight ring)
- Add `useRouter` from `next/navigation` (already imported elsewhere in the codebase)
- Replace `handleClick`:

  ```ts
  // Before:
  const handleClick = () => {
    if (!isViewerOpen) toggleViewer();
    setActiveItem(item);
  };

  // After:
  const router = useRouter();
  const handleClick = () => {
    router.push(`${classId}/${item.type}/${item.id}`);
  };
  ```

- Remove the `isActive` ring styling from the card div (`isActive && "ring-2 ring-blue-500 border-blue-200 shadow-sm"`)
- Keep the rest of the component (badges, icons, teacher dropdown, student "View" button, dialogs) unchanged

### 2. `resizable-content.tsx` — Simplify to single-column layout

- Remove imports: `ResizableHandle`, `ResizablePanel`, `ResizablePanelGroup`, `useDocumentViewer`, `ViewerToggle`, `DocumentViewerPanel`
- Replace the `ResizablePanelGroup` with a simple `<div>` that renders only the lesson list (left panel content)
- Remove the right panel (`DocumentViewerPanel`) and the `ResizableHandle`
- Remove the `isViewerOpen` check from `defaultSize`

### 3. `ClassOverviewClient/index.tsx` — Remove provider and mobile sheet

- Remove imports: `DocumentViewerProvider`, `ResizableContent`, `MobileDocumentSheet`
- Replace `DocumentViewerProvider` wrapper with a plain `<div>` or fragment
- Remove `<MobileDocumentSheet />` component
- Import the simplified content component (renamed from `ResizableContent` or inlined)

### 4. Delete unused files

These files are no longer needed after removing the viewer:

- `context.tsx` — `DocumentViewerProvider` / `useDocumentViewer`
- `document-viewer-panel.tsx` — Desktop inline viewer panel
- `mobile-document-sheet.tsx` — Mobile overlay viewer
- `viewer-toggle.tsx` — Already returns `null`, fully unused

### 5. No changes needed

- `types.ts` — Still needed for `Lesson`, `LessonType`, `LessonTypeKey`, `buildInitialData`, etc.
- ` topic-section.tsx` — Still renders `LessonTypeRow` items, no viewer dependency
- `empty-state.tsx` — No viewer dependency
- `ClassOverview.tsx` (server component) — No changes
- Announcement tab, Gradebook, Student tab — Unrelated
- Route pages (`handout/[lessonTypeId]`, `quiz/[lessonTypeId]`, `assignment/[lessonTypeId]`) — Already exist, no changes
