# Plan: Make TabsList Mobile Responsive in ClassIdView.tsx

## Goal

Make the `TabsList` (sidebar navigation) visible on mobile while maintaining usability.

## Current State

- The sidebar wrapper div has `hidden md:flex` - hiding it on mobile
- TabsList is vertical with `flex-col` layout
- TabsList has `h-full` (full height of parent)

## Proposed Changes

### 1. Make Sidebar Wrapper Visible on Mobile

Remove `hidden` from the wrapper div so it's always visible:

```
hidden md:flex md:flex-col md:w-56 md:h-full bg-sidebar/92
```

→

```
flex md:flex-col w-full md:w-56 h-auto md:h-full bg-sidebar/92
```

Key changes:

- Remove `hidden` - always show
- Change `h-full` to `h-auto` - on mobile, let height fit content
- Add `w-full` - full width on mobile

### 2. Make TabsList Horizontal on Mobile

Add mobile-specific styling to TabsList:

```
flex flex-col h-full w-full items-stretch justify-start rounded-none bg-transparent p-2 gap-1
```

→

```
flex flex-col md:flex-col h-auto md:h-full w-full items-stretch md:items-stretch justify-start rounded-none bg-transparent p-2 gap-1
```

Or alternatively, use responsive classes:

```
flex flex-col md:flex-col h-auto md:h-full w-full items-center md:items-stretch justify-start rounded-none bg-transparent p-2 gap-1
```

### 3. Adjust TabsTrigger Styling for Horizontal Layout

The TabsTrigger items may need width adjustments for horizontal layout:

- Add `w-full md:w-auto` or similar to make them full-width on mobile

### 4. Ensure Tabs Component Works with Horizontal Sidebar

The `orientation="vertical"` on Tabs may need to be conditionally set or removed for mobile.

## Alternative Approach

If the vertical sidebar approach doesn't work well on mobile, consider:

- Using a bottom tab bar pattern for mobile
- Using a collapsible drawer/sheet for mobile

## Files to Modify

- `src/modules/user/ui/Views/ClassIdView.tsx`

## Design Decision

- **Tab Position on Mobile**: Top (horizontal) - user confirmed

## Implementation Steps

### 1. Create Two Separate TabsList Components

The vertical sidebar and horizontal mobile tabs have different layouts. Use conditional rendering:

- Mobile: Horizontal TabsList at top
- Desktop: Vertical sidebar (existing layout)

### 2. Modify Wrapper Div for Sidebar

```
hidden md:flex md:flex-col md:w-56 md:h-full bg-sidebar/92
```

- Keep `hidden md:flex` for desktop sidebar-only visibility

### 3. Add Horizontal Mobile TabsList

Create a new `div` for mobile with horizontal TabsList:

```
className="flex md:hidden w-full bg-sidebar/92 border-b border-border"
```

Inside:

```
<TabsList className="flex w-full h-14 items-center justify-start overflow-x-auto gap-1 bg-transparent p-0">
```

### 4. Style TabsTrigger for Mobile Horizontal Layout

On each TabsTrigger, update className:

- Add `flex-shrink-0` to prevent shrinking
- Adjust padding for horizontal layout
- Keep visual consistency with desktop

### 5. Remove Orientation Conflict

The `orientation="vertical"` on Tabs may need conditional handling or removal for mobile. Test and verify TabsContent displays correctly.
