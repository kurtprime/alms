# Fix Gradebook Responsiveness

## Problem

The Gradebook table on desktop view has a typo and incorrect width calculation:

- Line 639: `w-sceen md:w-[calc(100vw-340px)]` has typo `w-sceen` instead of `w-screen`
- The width calculation `calc(100vw-340px)` doesn't properly account for the sidebar layout

## Analysis

The Gradebook is rendered inside the ClassIdView Tabs component:

- Desktop sidebar (TabsList): `w-56` (224px)
- Content area: `flex-1` (fills remaining space)

The table uses `ScrollArea` with manual width calculation that:

1. Has a typo breaking it entirely
2. Uses a hardcoded pixel value that may not match actual layout

## Solution

1. **Fix the typo**: Change `w-sceen` to `w-screen`
2. **Use flex-based width**: Instead of fixed pixel calculation, use `w-full` to let the ScrollArea fill its container, and constrain the inner Table with `min-w-full`
3. **Remove unnecessary width constraints**: The ScrollArea should just be `w-full` and let the table scroll horizontally as needed

## Changes to `src/modules/user/ui/components/Teacher/GradebookDataTable/index.tsx`

Line 639 change:

```jsx
// Before (broken):
<ScrollArea className="whitespace-nowrap w-sceen md:w-[calc(100vw-340px)]">

// After (fixed):
<ScrollArea className="w-full">
```

The Table inside will naturally overflow horizontally and the ScrollArea will handle scrolling properly.
