# Plan: Fix Vertical Tabs Width and Sidebar Toggle in ClassIdView

## Problems

1. **Tabs width**: Vertical tabs have fixed `md:w-[180px]` but should stretch to fill the entire left side
2. **Sidebar toggle broken**: The `useEffect` has `setOpen` in its dependency array. Since `setOpen` is recreated when `open` changes, the effect can re-run, calling `setOpen(false)` again and overriding the user's toggle action.

## Solutions

### 1. Fix tabs width in `ClassIdView.tsx`

Change `TabsList` class from `md:w-[180px]` to `md:flex-1`.

### 2. Fix sidebar toggle in `ClassIdViewWrapper.tsx`

Use `useRef` to track initialization and `previousState` so the effect only runs once on mount, not when `setOpen` identity changes.

**Current (buggy) code:**

```javascript
useEffect(() => {
  const previousState = document.cookie... // captured in closure
  setOpen(false);
  return () => { /* restore previousState */ };
}, [setOpen]); // setOpen changes when open changes → effect re-runs → setOpen(false) called again
```

**Fixed code:**

```javascript
useEffect(() => {
  if (hasInitializedRef.current) return; // prevent re-runs
  hasInitializedRef.current = true;

  previousStateRef.current = document.cookie... // store in ref
  setOpen(false);
  return () => { /* restore previousStateRef.current */ };
}, [setOpen]); // effect body won't re-run, only cleanup on unmount
```

This ensures:

- `setOpen(false)` is only called once on mount
- User can toggle sidebar open/close freely while on the page
- Previous state is restored when navigating away

## Summary of Changes

1. **`ClassIdView.tsx`**: `md:w-[180px]` → `md:flex-1` on TabsList
2. **`ClassIdViewWrapper.tsx`**: Add `useRef` guards to prevent effect re-runs
